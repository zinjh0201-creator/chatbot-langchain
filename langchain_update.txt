5. 메모리 연동 로직 - 완결
=============================================
✅ RunnableWithMessageHistory 기반 메모리 관리
   
   구현 구조:
   1. SessionChatMessageHistory: 세션별 메시지 저장
      - _get_session_history(session_id): 세션 히스토리 조회
      
   2. RunnableWithMessageHistory: LCEL 체인에 자동 메모리 주입
      - input_messages_key="input": 사용자 입력 키
      - history_messages_key="history": 메모리 키
      - get_session_history: 히스토리 콜백 함수
      
   3. MessagesPlaceholder(variable_name="history"): 프롬프트 템플릿에 히스토리 삽입
      
   4. 자동 히스토리 요약 (_summarize_session_history)
      - 메시지 20개 초과 시 최근 20개만 유지
      - 토큰 사용량 최적화
   
   실행 흐름:
   User Question
       ↓
   _get_session_history() → SessionChatMessageHistory 조회
       ↓
   MessagesPlaceholder에 history 자동 주입
       ↓
   LCEL Chain (프롬프트 → LLM)
       ↓
   Response
       ↓
   history_obj.add_message(AIMessage(response))
       ↓
   _summarize_session_history() → 20개 초과 시 정리

✅ 메모리 동기화 검증
   - 프롬프트 로깅에서 사용하는 history_obj.messages
   - LCEL에서 MessagesPlaceholder(variable_name="history")로 주입되는 메시지
   - 두 가지가 동일한 소스에서 가져와지므로 완벽히 동기화됨

✅ LCEL 체인에 메모리 주입 메커니즘 (코드 분석)
   
   핵심 코드:
   ──────────────────────────────────────────────────
   # 1. 프롬프트 템플릿 정의
   prompt_template = ChatPromptTemplate.from_messages([
       ("system", "...시스템 프롬프트..."),
       MessagesPlaceholder(variable_name="history"),  # ← 히스토리 삽입점
       ("human", "{input}"),
   ])
   
   # 2. LCEL 체인 구성
   chain = (
       {
           "context": RunnableLambda(lambda _: context),
           "history": RunnableLambda(lambda x: x.get("history", [])),
           "input": RunnablePassthrough()
       }
       | prompt_template
       | self.llm
   )
   
   # 3. RunnableWithMessageHistory로 래핑
   runnable_with_history = RunnableWithMessageHistory(
       runnable=chain,
       get_session_history=self._get_session_history,  # ← 메모리 가져오기
       input_messages_key="input",
       history_messages_key="history",  # ← "history" 키로 주입
   )
   ──────────────────────────────────────────────────
   
   실행 흐름:
   1. 사용자가 question 입력
   2. RunnableWithMessageHistory가 config에서 session_id 추출
   3. get_session_history(session_id)를 호출하여 SessionChatMessageHistory 획득
   4. SessionChatMessageHistory.messages에서 모든 메시지 추출
   5. {"history": messages} 형태로 LCEL에 주입
   6. MessagesPlaceholder(variable_name="history")가 이 메시지들을 프롬프트에 삽입
   7. 최종 프롬프트: [SYSTEM] + [HISTORY_MESSAGES] + [HUMAN_INPUT]로 LLM에 전달
   
   ✅ 메모리 주입 검증:
      - get_session_history 콜백: 세션별로 정확한 히스토리 반환
      - history_messages_key="history": 데이터가 정확한 키로 주입됨  
      - MessagesPlaceholder 자동 처리: 메시지 객체가 프롬프트에 올바르게 포함됨
