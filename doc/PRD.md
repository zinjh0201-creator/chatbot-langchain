# PRD (Product Requirements Document) — RAG Chatbot

## 1. 개요
- 프로젝트명: RAG Chatbot (Supabase pgvector + Gemini)
- 목표: PDF 문서를 업로드하여 문서 기반 질문 응답을 제공하고, 문서 관련 정보가 없을 때는 Gemini LLM을 통한 답변을 제공하는 웹 애플리케이션.
- 사용자: 사내 임직원, 고객 지원 담당자, 지식 검색 사용자.

## 2. 핵심 기능
1. 문서 업로드
   - PDF 업로드(DnD/파일 선택)
   - 최대 200MB 제한
2. RAG 질문 응답
   - 백엔드에서 Supabase pgvector 유사도 검색
   - 유사도 ≥ 0.7 시 문서참조답변
   - 유사도 < 0.7 시 Gemini 추론답변
3. 채팅 UI
   - 사용자 질문/봇 답변 저장
   - 채팅 히스토리 로컬스토리지 보존
   - 히스토리 삭제 버튼
4. 상태표시
   - 업로드 진행, 오류
   - 질문 응답 대기 중 로딩 애니메이션

## 3. 비기능 요구사항
- 보안
  - Gemini API 키, DB URL을 서버 측 환경변수로 관리
  - CORS 허용 대상 제한(배포 도메인)
- 성능
  - 클라이언트 최소 자바스크립트 번들
  - 백엔드 커넥션 풀 사용
- 배포
  - 프론트엔드: Vercel
  - 백엔드: 서버리스/VM (적절히 설정)

## 4. 테스트 케이스
### TC1 — PDF 업로드
- 조건: 유효한 PDF 업로드
- 예상: 업로드 완료, 문서가 DB에 삽입

### TC2 — 과거 QA 히스토리
- 조건: 입력한 질문 후 새로고침
- 예상: 채팅 기록 로컬 저장 후 복원

### TC3 — API 실패 처리
- 조건: 백엔드 접속 불가
- 예상: 사용자에게 에러 메시지 표시

## 5. 향후 확장
data augmentation, 문서 다국어 지원, 이미 업로드된 문서 검색, 인증/권한, 엑셀/워드 변환, 대용량 배치 인제스트.
